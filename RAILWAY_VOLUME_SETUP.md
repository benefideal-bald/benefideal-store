# Настройка персистентного хранилища для базы данных на Railway

## Проблема
База данных SQLite стирается при каждом деплое, потому что файловая система Docker контейнера не персистентна.

## Решение: Railway Volume

### Шаг 1: Создайте Volume в Railway
1. Откройте ваш проект на Railway: https://railway.app
2. Выберите ваш сервис (benefideal-store)
3. Перейдите в раздел **"Volumes"** (или **"Storage"**)
4. Нажмите **"New Volume"**
5. Назовите его: `database-storage`
6. Укажите путь монтирования: `/app/data`
7. Нажмите **"Add"**

### Шаг 2: Настройте переменную окружения
1. В разделе **"Variables"** вашего сервиса
2. Добавьте переменную:
   - **Key:** `DATABASE_PATH`
   - **Value:** `/app/data/subscriptions.db`

### Шаг 3: Перезапустите сервис
После создания Volume перезапустите сервис, чтобы изменения вступили в силу.

## Альтернативное решение: PostgreSQL
Если Volume не работает, используйте PostgreSQL:
1. Создайте PostgreSQL базу данных на Railway
2. Измените код для использования PostgreSQL вместо SQLite
3. База данных будет полностью персистентной

## Проверка
После настройки проверьте логи:
- `✅ Database opened successfully at: /app/data/subscriptions.db`
- `✅ Database file exists: true`

Заказы должны сохраняться между деплоями.
