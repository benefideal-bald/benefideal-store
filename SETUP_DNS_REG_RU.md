# Настройка DNS на REG.RU для Railway

## Шаг 1: Войти в панель управления REG.RU
1. Откройте https://www.reg.ru
2. Войдите в личный кабинет
3. Перейдите в раздел **"Домены"** → выберите `benefideal.ru`

## Шаг 2: Настроить DNS записи
1. В карточке домена нажмите **"Управление DNS"** или **"DNS-серверы и зона"**
2. Выберите **"Использовать DNS-серверы REG.RU"** (если еще не выбрано)
3. Перейдите в раздел **"DNS-записи"** или **"Редактировать зону"**

## Шаг 3: Добавить CNAME запись
1. Нажмите **"Добавить запись"** или **"+"**
2. Выберите тип: **CNAME**
3. Заполните:
   - **Имя/Поддомен:** `@` (или оставьте пустым для корневого домена)
   - **Значение/Каноническое имя:** `9gv052rd.up.railway.app`
   - **TTL:** `3600` (или оставьте по умолчанию)
4. Нажмите **"Сохранить"** или **"Добавить"**

## Шаг 4: Проверить существующие записи
⚠️ **ВАЖНО:** Если есть запись типа **A** для `@` (корневого домена), её нужно **удалить** или изменить, так как CNAME не может сосуществовать с A-записью для того же имени.

## Шаг 5: Сохранить изменения
1. Нажмите **"Сохранить"** или **"Применить изменения"**
2. Подождите 5-30 минут (обычно обновляется быстро)

## Шаг 6: Проверить
После настройки проверьте:
```bash
dig benefideal.ru CNAME
```
Должен вернуть: `9gv052rd.up.railway.app`

Или просто откройте: https://benefideal.ru
Должен работать через Railway.

## Если не получается добавить CNAME для @
На REG.RU иногда нельзя добавить CNAME для корневого домена (@). В этом случае:
1. Используйте **A-запись** вместо CNAME
2. Railway должен показать IP-адрес (спросите в Railway поддержке или проверьте документацию)
3. Или используйте поддомен: `www.benefideal.ru` → CNAME → `9gv052rd.up.railway.app`

## После настройки
- Railway автоматически выдаст SSL сертификат
- Статус в Railway изменится на "Active"
- `benefideal.ru` будет работать через Railway
- Заказы будут отображаться в админ-панели
