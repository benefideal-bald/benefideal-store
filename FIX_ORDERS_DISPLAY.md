# Исправление отображения заказов

## Решение
Заказы теперь сохраняются в `orders.json` (как отзывы в `reviews.json`), что гарантирует их сохранение при деплое.

4. ✅ Обновлен `/api/admin/orders` - читает из JSON + базы данных

1. Проверьте консоль браузера (F12) на наличие ошибок
2. Проверьте, что API возвращает заказы (см. команду выше)
3. Убедитесь, что Railway задеплоил последние изменения

## Важно:
- Заказы теперь хранятся в Git (в `orders.json`), как отзывы
- При каждом новом заказе он автоматически добавляется в `orders.json`
- Заказы не потеряются при деплое, потому что файл в Git
